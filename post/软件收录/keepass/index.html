不是危言耸听，目前的密码泄露事件太多了，就算你能保证自己没有泄露出自己的密码给第三方，但现在的好多商业公司根本就不当用户隐私是一回事，连数据库可以公开访问已经见怪不怪了。

简介

既然商业平台都无法信任，那就只考虑开源的软件，毫无疑问就是Keepass，无论是安全的程度，还是多平台同步的支持，都是排在前列的。

下面是官方的介绍：

KeePass is a free open source password manager, which helps you to manage your passwords in a secure way. You can put all your passwords in one database, which is locked with one master key or a key file. So you only have to remember one single master password or select the key file to unlock the whole database. The databases are encrypted using the best and most secure encryption algorithms currently known (AES and Twofish).

下载安装

官方下载地址：https://keepass.info/download.html

翻译文件下载（windows系统解压到C:\Program Files (x86)\KeePass Password Safe 2\Languages，选择change language，重启keepass ）：https://keepass.info/translations.html

一般在各大软件源也有下载，如果是比较重要的密码建议阅读源码并从源码编译安装，并采取其他措施（开源只是表明软件不包含恶意代码，并不能保证没有安全漏洞，更多信息请参考这里）。

值得注意的是，keepass有很多第三方开发的客户端，一般也是开源的，在官方下载页下方就有收录。正因如此，才使keepass的多平台使用体验变得越来越好，感谢开源！

keepass所提供密码加强的途径

  • 网络隔离

  • 完全或几乎（软件本身更新，可以禁止）离线，软件本身权限控制在最小范围。

  • 默认情况下，KeePass数据库存储在本地文件系统上（而不是云存储）。

  • 文件加密：可以通过主密码，密钥文件和当前Windows帐户详细信息的任意组合来保护KeePass数据库文件。

  • 主密码：高强度的主密码可以让数据库暴力破解的可能性几乎为0。

  • 密钥文件：文件名和类型都没有限制，增加了破解难度。

  • windows用户账户：使用windows用户账户的数据加密，这使得在另一台电脑上解密数据库几乎成为不可能，包括重装系统，虽然比较安全但要慎用。

  • 密码强度

  • 随机生成。

  • 支持定期更换。

  • 密码转移

  • 输入密码时启动安全桌面模式，一定程度上预防系统木马盗取密码。

  • 定时清理剪贴板。

  • 丰富而开源的插件，提供二步验证等功能，你甚至可以自己开发一个。

一些注意点

  • 密码数据库和key文件如果使用网盘自动备份，请不要同时上传到一个网盘。
  • 不用时及时关闭软件，运行中的软件更容易受到攻击导致密码泄露。

安全软件的矛盾点

易用性和安全性的矛盾

易用往往意味着代码量的增加，运行中的漏洞出现的可能性也就增大。

安全与备份的矛盾

一个密码最安全的情况就是没有存储，也就是不存在于任何地方，但这是不可能的，目前最安全的存储地是我们的大脑。但是我们的记忆能力是有限的，所以才需要一个第三方的密码管理软件。因此密码管理软件就肩负了两个相互矛盾的使命：保存密码，保证密码的安全性。

脆弱的宿主系统以及硬件漏洞

如果宿主系统或者硬件出现了漏洞，那密码管理软件无论怎么做，都无济于事。

用户的安全意识高低

用户的安全意识如果足够高的话，密码管理软件就可以做的很简单，不需要做出一些本该由用户做且用户会做的更好的事情，这样也就能专注于安全性了。
