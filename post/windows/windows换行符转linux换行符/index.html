Windows 和 Linux 有数不尽的护城河，在两个系统之间操作的时候，就会踩到很多坑，比如本文要讲的换行符，当你写一个Shell脚本或其他文件上传到服务器运行时，提示 $'\r': command not found的时候，就是踩到这个坑了。

在windows里操作

一些文本编辑软件会附带这个功能。以notepad++为例：

  1. 点击菜单【编辑】
  2. 拖动鼠标到【文档格式转换】
  3. 点击【转换为Unix (LF)】

在Linux里操作

使用VIM

vim test.txt打开文件后，先按:键，输入命令set ff=unix，回车运行，:wq保存即可。

sed

1

sed -i 's/\r//g' test.txt

#此命令在Windows系统中若安装了git，在git bash里会附带，可以在Windows系统的git bash里直接运行
