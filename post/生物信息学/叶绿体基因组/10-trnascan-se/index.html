10 TRNAscan SE

2023-07-10
生信软件
约 872 字 预计阅读 2 分钟 次阅读

文章目录

    • 运行过程
    • 参数
    • 参考

一行命令开始：

tRNAscanSE能在基因组水平上进行tRNA扫描，该软件利用perl整合了3个独立的tRNA检测软件：

  • tRNAscan
  • EufindRNA
  • Cove

运行过程

  1. 调用tRNAscan和EufindRNA鉴定基因组序列中的tRNA区域，做第一遍的初步筛选，得到候选 tRNA，在stats结果中会有一个First-pass Stats的统计结果，我理解的就是对应的初步筛选的结果。
  2. 通过Cove做进一步的验证，这样既保证了前者的sensitivities， 又保证了后者较低的假阳性概率，同时在搜索速度上提升了很多。

参数

-X –score: set cutoff score (in bits) for reporting tRNAs (default=20)，判定为tRNAs的阈值。

-z –pad: usenucleotides padding when passing first-pass

tRNA bounds predictions to CM analysis (default=8)

-o –output: save final results in，输出文件。

-f –struct: save tRNA secondary structures to输出保留了tRNA二级结构的文件。

-s –isospecific: save results using isotype-specific models in

-m –stats: save statistics summary for run in，统计结果文件。

-b –bed: save results in BED file format of，输出bed格式结果。

-j –gff: save results in GFF3 file format of，输出gff3格式结果。

有的人使用的版本比较老，当时还没有这个参数可以直接生成gff文件，需要另外的脚本程序转gff文件。

perl convert_tRNAScanSE_to_gff3.pl -i tRNA.out >tRNAscan.gff3

PS：convert_tRNAScanSE_to_gff3.pl 可参考：github: https://github.com/jorvis/biocode/blob/master/gff/convert_tRNAScanSE_to_gff3.pl

–thread，线程数。

-c –conf: tRNAscan-SE configuration file (default: tRNAscan-SE.conf)，指定tRNAscan-SE.conf文件。

其它参数：

-E：适用于真核生物，默认。

-A：适合于古细菌。该参数选择了古细菌特异性的covariance model(cm)，同时稍微放宽了 EufindtRNA 的 cutoffs。

-B：适合于细菌。

-G：适合于古细菌，细菌和真核生物的混合序列。该参数使用 general tRNA covariance model。

-O：适合于线粒体和叶绿体。选择该参数，则仅使用 Cove 进行分析，搜索速度会很慢，同时也不能给出 pseudogenes 检测。

-C 仅使用 Cove 进行 tRNA 分析。虽然从一定程度上提高了准确性，但是会极慢，当然不建议了。

参考

improved detection and functional classification of transfer RNA genes”,

Nucleic Acids Res. 49:9077–9096.
